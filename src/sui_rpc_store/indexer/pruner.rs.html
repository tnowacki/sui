4//! Background pruner for the historical column families.
5//!
6//! Pruning is a standalone [`Service`] rather than a framework
7//! pipeline: it does not consume checkpoints from ingestion, it reads
8//! the already-committed state and deletes data below a retention
9//! floor. The shape mirrors the validator's perpetual-store pruner
10//! (a periodic background task) more than the indexer framework's
11//! per-pipeline `prune` hook — the deletions are data-driven (we walk
12//! transaction effects to retract superseded object versions) and the
13//! floor is a single value shared across every historical CF.
14//!
15//! # What gets pruned
16//!
17//! - **Per-transaction CFs** (`transactions`, `effects`, `events`,
18//!   `tx_metadata_by_seq`) — range-deleted over the pruned `tx_seq`
19//!   range; the keys are contiguous big-endian `tx_seq`, so one range
20//!   tombstone per CF clears the chunk.
21//! - **Per-checkpoint CFs** (`checkpoint_summary`,
22//!   `checkpoint_contents`) — range-deleted over the pruned
23//!   checkpoint range.
24//! - **Digest reverse indexes** (`tx_seq_by_digest`,
25//!   `checkpoint_seq_by_digest`) — point-deleted; their keys are
26//!   digests, so we collect them from the data being pruned (tx
27//!   digests from each effects row, checkpoint digests from each
28//!   summary) before deleting.
29//! - **`objects` history** — point-deleted, effects-driven: each
30//!   pruned transaction's `modified_at_versions` (superseded input
31//!   versions) and `all_tombstones` (deleted / wrapped markers) are
32//!   the exact `(ObjectID, version)` rows that are now dead. The
33//!   latest live version is never an input to a pruned transaction,
34//!   so it — and the `live_objects` pointer at it — is preserved.
35//! - **Ledger-history bitmaps** (`transaction_bitmap`,
36//!   `event_bitmap`) — not deleted directly; advancing the shared
37//!   [`tx_seq_floor`](crate::schema::pruning_watermark::tx_seq_floor)
38//!   lets their compaction filters drop fully-pruned buckets. We
39//!   force a compaction once the floor advances so the eviction is
40//!   prompt rather than waiting for a natural sweep.
41//!
42//! The live-set-bounded indexes (`live_objects`, `object_by_owner`,
43//! `object_by_type`, `balance`, `package_versions`) and the tiny
44//! `epochs` CF are never pruned.
45//!
46//! # Floor, retention, and safety
47//!
48//! Retention is epoch-based: the `retention_epochs` most-recent
49//! epochs are retained, and the target floor is the start checkpoint
50//! of the oldest retained epoch. The floor is then clamped so it
51//! never advances past the oldest in-memory snapshot's checkpoint:
52//! point and range deletes are already invisible to a snapshot
53//! (RocksDB pins the data a live snapshot references), but the bitmap
54//! compaction filter physically removes buckets irrespective of
55//! snapshots, so the clamp keeps every live snapshot's advertised
56//! available range valid even under an aggressively small retention.
57//!
58//! Each tick advances the floor toward that target by at most
59//! `max_checkpoints_per_tick` checkpoints (in `max_chunk_checkpoints`
60//! atomic chunks), so a large backlog — for example when pruning is
61//! first enabled on an old database — drains across many ticks rather
62//! than one long blocking pass. The floor converges to the target
63//! over subsequent ticks.
64//!
65//! # Ordering and crash-safety
66//!
67//! Each chunk stages all of its deletes plus the new
68//! `PruningWatermarks` row into one atomic batch, commits, and only
69//! then advances the in-memory bitmap floor. Because the watermark
70//! row lives in the same batch as the deletes, a crash either loses
71//! the whole chunk (re-pruned next run) or commits it wholesale;
72//! there is no partial-delete-without-watermark state. Range and
73//! point deletes are idempotent, so a re-run is harmless.

153/// Start the background pruner as a [`Service`].
154///
155/// Errors if `config.retention_epochs` is `0` (which would prune the
156/// current epoch). The returned service runs an infinite tick loop;
157/// it is aborted on graceful shutdown (each chunk is atomic, so an
158/// abort leaves the database consistent).
159pub fn start_pruner(
160    db: Db,
161    config: PrunerConfig,
162    metrics: Arc<PrunerMetrics>,
163) -> anyhow::Result<Service> {
164    anyhow::ensure!(
165        config.retention_epochs >= 1,
166        "PrunerConfig::retention_epochs must be >= 1; 0 would prune the current epoch",
167    );
168    anyhow::ensure!(
169        config.max_checkpoints_per_tick >= 1,
170        "PrunerConfig::max_checkpoints_per_tick must be >= 1; 0 would never make progress",
171    );
172
173    // Reconstruct typed handles once; they are cheap views over the
174    // shared `Db` and are reused across every tick.
175    let schema = Arc::new(RpcStoreSchema::open(&db).context("Opening schema for pruner")?);
176
177    let service = Service::new().spawn_aborting(async move {
178        let mut ticker = tokio::time::interval(config.interval());
179        ticker.set_missed_tick_behavior(MissedTickBehavior::Delay);
180
181        loop {
182            ticker.tick().await;
183
184            let db = db.clone();
185            let schema = schema.clone();
186            let config = config.clone();
187            let metrics = metrics.clone();
188
189            // The pruner does blocking RocksDB iteration and writes;
190            // keep it off the async runtime threads.
191            let res =
192                tokio::task::spawn_blocking(move || prune_once(&db, &schema, &config, &metrics))
193                    .await;
194
195            match res {
196                Ok(Ok(())) => {}
197                Ok(Err(e)) => {
198                    warn!("rpc-store pruner pass failed (will retry next interval): {e:#}")
199                }
200                Err(e) => warn!("rpc-store pruner task join error: {e}"),
201            }
202        }
203    });
204
205    Ok(service)
206}

208/// Run a single pruning pass: recompute the target floor and advance
209/// the persisted floor toward it one chunk at a time.
210fn prune_once(
211    db: &Db,
212    schema: &RpcStoreSchema,
213    config: &PrunerConfig,
214    metrics: &PrunerMetrics,
215) -> anyhow::Result<()> {
216    let Some(current_epoch) = current_committed_epoch(db)? else {
217        debug!("rpc-store pruner: no committed watermark yet; nothing to prune");
218        return Ok(());
219    };
220
221    let Some(retention_lo) =
222        retention_checkpoint_floor(schema, current_epoch, config.retention_epochs)?
223    else {
224        debug!(
225            current_epoch,
226            "rpc-store pruner: retention floor not yet reached; nothing to prune"
227        );
228        return Ok(());
229    };
230
231    // Never advance the floor past the oldest live snapshot.
232    let target_lo = clamp_to_snapshot(retention_lo, db.snapshot_range().map(|r| *r.start()));
233
234    let mut cursor = schema.get_pruning_watermarks()?.unwrap_or_default();
235    if target_lo <= cursor.checkpoint_lo {
236        debug!(
237            target_lo,
238            current_lo = cursor.checkpoint_lo,
239            "rpc-store pruner: floor already at or beyond target"
240        );
241        return Ok(());
242    }
243
244    // Bound the work done this tick: advance the floor by at most
245    // `max_checkpoints_per_tick` checkpoints so a large backlog drains
246    // across many ticks instead of one long blocking pass. The floor
247    // converges to `target_lo` over subsequent ticks.
248    let tick_target = target_lo.min(cursor.checkpoint_lo + config.max_checkpoints_per_tick);
249
250    info!(
251        from = cursor.checkpoint_lo,
252        to = tick_target,
253        target = target_lo,
254        current_epoch,
255        "rpc-store pruner: advancing floor"
256    );
257
258    while cursor.checkpoint_lo < tick_target {
259        let chunk_ckpt_hi = (cursor.checkpoint_lo + config.max_chunk_checkpoints).min(tick_target);
260        cursor = prune_chunk(db, schema, cursor, chunk_ckpt_hi, metrics)?;
261        metrics.checkpoint_lo.set(cursor.checkpoint_lo as i64);
262        metrics.tx_seq_lo.set(cursor.tx_seq_lo as i64);
263        metrics.chunks_committed.inc();
264    }
265
266    // The bitmap CFs' compaction filters only drop fully-pruned
267    // buckets on a compaction sweep; force one once the floor has
268    // reached its retention target so the eviction is prompt. While a
269    // backlog is still draining over multiple ticks we skip the
270    // whole-CF compaction so it does not become the per-tick long
271    // pole; natural background compaction still applies the same
272    // filter opportunistically in the meantime, and the final
273    // catch-up tick forces a prompt sweep.
274    if cursor.checkpoint_lo >= target_lo {
275        db.compact_range_cf(transaction_bitmap::NAME, None, None)
276            .context("Compacting transaction_bitmap after prune")?;
277        db.compact_range_cf(event_bitmap::NAME, None, None)
278            .context("Compacting event_bitmap after prune")?;
279    }
280
281    Ok(())
282}

284/// Prune one chunk of checkpoints `[cursor.checkpoint_lo,
285/// chunk_ckpt_hi)` and their transactions, returning the new floor.
286fn prune_chunk(
287    db: &Db,
288    schema: &RpcStoreSchema,
289    cursor: Watermarks,
290    chunk_ckpt_hi: u64,
291    metrics: &PrunerMetrics,
292) -> anyhow::Result<Watermarks> {
293    let ckpt_lo = cursor.checkpoint_lo;
294    let tx_lo = cursor.tx_seq_lo;
295
296    // The exclusive `tx_seq` upper bound for the chunk is the
297    // cumulative network tx count after the chunk's highest
298    // checkpoint, which is the first `tx_seq` of `chunk_ckpt_hi`.
299    // `chunk_ckpt_hi >= 1` by the caller's loop invariant, and
300    // `chunk_ckpt_hi - 1 >= ckpt_lo` is still retained (not yet
301    // pruned), so its summary is present.
302    let last_ckpt = chunk_ckpt_hi - 1;
303    let tx_hi = schema
304        .get_checkpoint_summary(last_ckpt)?
305        .with_context(|| format!("checkpoint_summary missing for checkpoint {last_ckpt}"))?
306        .data()
307        .network_total_transactions;
308
309    let mut batch = db.batch();
310    let mut objects_deleted: u64 = 0;
311
312    // Transaction side. Each effects row yields the object versions
313    // to retract and the transaction digest to unindex. A missing
314    // row means the range was already pruned (idempotent re-run).
315    for tx_seq in tx_lo..tx_hi {
316        let Some((effects, _unchanged)) = schema.get_effects(tx_seq)? else {
317            continue;
318        };
319        for (id, version) in effects.modified_at_versions() {
320            batch.delete(&schema.objects, &objects::Key { id, version })?;
321            objects_deleted += 1;
322        }
323        for (id, version) in effects.all_tombstones() {
324            batch.delete(&schema.objects, &objects::Key { id, version })?;
325            objects_deleted += 1;
326        }
327        batch.delete(
328            &schema.tx_seq_by_digest,
329            &tx_seq_by_digest::Key(*effects.transaction_digest()),
330        )?;
331    }
332
333    // The `tx_seq`-keyed CFs are contiguous, so one range delete each
334    // clears the whole chunk regardless of how many rows it spans.
335    batch.delete_range(&schema.transactions, &U64Be(tx_lo), &U64Be(tx_hi))?;
336    batch.delete_range(&schema.effects, &U64Be(tx_lo), &U64Be(tx_hi))?;
337    batch.delete_range(&schema.events, &U64Be(tx_lo), &U64Be(tx_hi))?;
338    batch.delete_range(&schema.tx_metadata_by_seq, &U64Be(tx_lo), &U64Be(tx_hi))?;
339
340    // Checkpoint side. Collect each checkpoint's digest for the
341    // reverse index, then range-delete the seq-keyed CFs.
342    for seq in ckpt_lo..chunk_ckpt_hi {
343        let Some(summary) = schema.get_checkpoint_summary(seq)? else {
344            continue;
345        };
346        batch.delete(
347            &schema.checkpoint_seq_by_digest,
348            &checkpoint_seq_by_digest::Key(summary.data().digest()),
349        )?;
350    }
351    batch.delete_range(
352        &schema.checkpoint_summary,
353        &U64Be(ckpt_lo),
354        &U64Be(chunk_ckpt_hi),
355    )?;
356    batch.delete_range(
357        &schema.checkpoint_contents,
358        &U64Be(ckpt_lo),
359        &U64Be(chunk_ckpt_hi),
360    )?;
361
362    // Advance the persisted floor atomically with the deletes.
363    let new = Watermarks {
364        tx_seq_lo: tx_hi,
365        checkpoint_lo: chunk_ckpt_hi,
366    };
367    let (k, v) = pruning_watermark::store(&new);
368    batch.put(&schema.pruning_watermark, &k, &v)?;
369
370    batch.commit()?;
371
372    // The commit is durable; advance the in-memory bitmap floor so
373    // the compaction filters drop buckets below `tx_hi`.
374    schema.set_pruning_floor(new.tx_seq_lo);
375    metrics.objects_deleted.inc_by(objects_deleted);
376
377    Ok(new)
378}

380/// Prune the embedded fullnode's history cohort up to a floor supplied
381/// by the validator's perpetual-store pruner.
382///
383/// Unlike [`start_pruner`], this is not epoch-driven and not a
384/// `Service`. The embedded deployment deactivates the raw chain-data
385/// CFs (`transactions`, `effects`, `events`, `objects`,
386/// `checkpoint_*`), so it can neither derive a retention floor nor walk
387/// effects to find the rows to delete. Instead the perpetual pruner —
388/// which owns the raw data — supplies the floor directly, and this
389/// prunes exactly the history-cohort CFs that grow without bound:
390///
391/// - `tx_metadata_by_seq` — range-deleted over
392///   `[old_tx_lo, pruned_tx_seq_exclusive)`.
393/// - `tx_seq_by_digest` — point-deleted; the digests are read from
394///   `tx_metadata_by_seq` (the only history CF that still carries them)
395///   over the pruned range, before that range is deleted.
396/// - `transaction_bitmap` / `event_bitmap` — evicted by advancing the
397///   shared `tx_seq` floor so their compaction filters drop
398///   fully-pruned buckets, then forcing a compaction.
399///
400/// The live cohort and the tiny `epochs` CF are never pruned.
401///
402/// `pruned_checkpoint_watermark` is the highest checkpoint the
403/// perpetual store has pruned (inclusive); `pruned_tx_seq_exclusive` is
404/// the first still-retained `tx_seq`. These mirror
405/// `sui_core::rpc_index::RpcIndexStore::prune`'s parameters so the
406/// embedded rpc-store and the legacy index prune in lockstep on the
407/// same floor. Idempotent: a re-run with the same or a lower floor is a
408/// no-op.
409pub fn prune_history_cohort(
410    db: &Db,
411    schema: &RpcStoreSchema,
412    pruned_checkpoint_watermark: u64,
413    pruned_tx_seq_exclusive: u64,
414) -> anyhow::Result<()> {
415    let cursor = schema.get_pruning_watermarks()?.unwrap_or_default();
416    let tx_lo = cursor.tx_seq_lo;
417    let tx_hi = pruned_tx_seq_exclusive;
418    // Lowest still-available checkpoint after this prune: the perpetual
419    // store has pruned through `pruned_checkpoint_watermark` inclusive.
420    let checkpoint_lo = pruned_checkpoint_watermark.saturating_add(1);
421
422    // No-op if the floor would not advance on either axis (idempotent
423    // re-run, or the perpetual floor is behind ours).
424    if tx_hi <= tx_lo && checkpoint_lo <= cursor.checkpoint_lo {
425        return Ok(());
426    }
427
428    let mut batch = db.batch();
429
430    // Unindex the digest reverse map for the pruned `tx_seq` range. The
431    // digests live in `tx_metadata_by_seq`; iterate it (seeking to the
432    // first present row) rather than point-getting each `tx_seq`, so a
433    // sparse range or an unknown (zero) floor costs work proportional to
434    // the rows present, not to the width of the interval.
435    for entry in schema.iter_tx_seq_digests(tx_lo, tx_hi)? {
436        let (_tx_seq, digest) = entry?;
437        batch.delete(&schema.tx_seq_by_digest, &tx_seq_by_digest::Key(digest))?;
438    }
439    batch.delete_range(&schema.tx_metadata_by_seq, &U64Be(tx_lo), &U64Be(tx_hi))?;
440
441    // Advance the persisted floor atomically with the deletes, taking
442    // the monotonic max on each axis so a stale lower floor never
443    // regresses an axis the other call already advanced.
444    let new = Watermarks {
445        tx_seq_lo: tx_hi.max(tx_lo),
446        checkpoint_lo: checkpoint_lo.max(cursor.checkpoint_lo),
447    };
448    let (k, v) = pruning_watermark::store(&new);
449    batch.put(&schema.pruning_watermark, &k, &v)?;
450    batch.commit()?;
451
452    // Durable now: advance the in-memory bitmap floor and force a
453    // compaction so the bitmap filters drop fully-pruned buckets
454    // promptly rather than waiting for a natural sweep.
455    schema.set_pruning_floor(new.tx_seq_lo);
456    db.compact_range_cf(transaction_bitmap::NAME, None, None)
457        .context("Compacting transaction_bitmap after prune")?;
458    db.compact_range_cf(event_bitmap::NAME, None, None)
459        .context("Compacting event_bitmap after prune")?;
460
461    Ok(())
462}

464/// The lowest epoch fully committed across every registered pipeline,
465/// or `None` if no pipeline has committed a watermark yet.
466///
467/// Taking the minimum is deliberately conservative: it lags the true
468/// tip epoch by at most one epoch while a pipeline catches up across
469/// a boundary, which only ever causes the pruner to retain slightly
470/// more.
471fn current_committed_epoch(db: &Db) -> anyhow::Result<Option<u64>> {
472    let framework = FrameworkSchema::new(db.clone());
473    let mut min_epoch: Option<u64> = None;
474    for entry in framework.watermarks.iter(..)? {
475        let (_, watermark) = entry?;
476        let epoch = watermark.epoch_hi_inclusive;
477        min_epoch = Some(min_epoch.map_or(epoch, |m| m.min(epoch)));
478    }
479    Ok(min_epoch)
480}

482/// The target checkpoint floor implied by epoch-based retention: the
483/// start checkpoint of the oldest epoch that is still retained.
484///
485/// Returns `None` when nothing is eligible yet — either the chain is
486/// younger than the retention window, or the oldest retained epoch's
487/// row (or its `start_checkpoint`) has not been observed.
488fn retention_checkpoint_floor(
489    schema: &RpcStoreSchema,
490    current_epoch: u64,
491    retention_epochs: u64,
492) -> anyhow::Result<Option<u64>> {
493    debug_assert!(retention_epochs >= 1, "validated in start_pruner");
494
495    // Retain epochs `[oldest_retained, current_epoch]`.
496    let oldest_retained = current_epoch.saturating_sub(retention_epochs - 1);
497    if oldest_retained == 0 {
498        // Epoch 0 is still retained, so no epoch has fully aged out.
499        return Ok(None);
500    }
501
502    let Some(info) = schema.get_epoch(oldest_retained)? else {
503        return Ok(None);
504    };
505    Ok(info.start_checkpoint)
506}

508/// Clamp the retention-derived floor so it never advances past the
509/// oldest in-memory snapshot's checkpoint. With no snapshots the
510/// retention floor stands; otherwise the floor is held at or below
511/// the oldest snapshot so that snapshot's advertised available range
512/// stays valid (and the bitmap compaction filter, which ignores
513/// snapshots, never drops a bucket the snapshot still serves).
514fn clamp_to_snapshot(retention_lo: u64, oldest_snapshot: Option<u64>) -> u64 {
515    match oldest_snapshot {
516        Some(snap) => retention_lo.min(snap),
517        None => retention_lo,
518    }
519}

705    /// The bitmap eviction path end to end: with the floor advanced
706    /// past a bucket, a forced compaction runs the bucket's
707    /// compaction filter and drops it, while a bucket above the floor
708    /// survives. This is what `prune_once` relies on when it compacts
709    /// the bitmap CFs after a floor advance.
710    #[test]
711    fn bitmap_buckets_below_floor_are_evicted_by_compaction() {
712        use std::sync::atomic::Ordering;
713
714        use crate::schema::pruning_watermark::tx_seq_floor;
715        use crate::schema::transaction_bitmap;
716
717        // The floor is process-wide; snapshot and restore it so this
718        // test doesn't perturb others sharing the same atomic.
719        let baseline = tx_seq_floor().load(Ordering::Relaxed);
720        let (_dir, db, schema) = fresh_db();
721        let dim = b"sender:alice".to_vec();
722
723        // Materialize one bucket fully below the floor (bucket 0) and
724        // one above it (bucket 1). The compaction filter keys off the
725        // bucket id in the key, so the stored bitmap contents are
726        // immaterial here.
727        let mut bitmap0 = roaring::RoaringBitmap::new();
728        bitmap0.insert(transaction_bitmap::bit_of(5));
729        let mut bitmap1 = roaring::RoaringBitmap::new();
730        bitmap1.insert(transaction_bitmap::bit_of(
731            transaction_bitmap::TX_BUCKET_SIZE + 5,
732        ));
733        let (k0, v0) = transaction_bitmap::store_bitmap(dim.clone(), 0, bitmap0);
734        let (k1, v1) = transaction_bitmap::store_bitmap(dim.clone(), 1, bitmap1);
735
736        let mut batch = db.batch();
737        batch.put(&schema.transaction_bitmap, &k0, &v0).unwrap();
738        batch.put(&schema.transaction_bitmap, &k1, &v1).unwrap();
739        batch.commit().unwrap();
740        db.flush().unwrap();
741
742        // Advance the floor to the top of bucket 0, then force a
743        // compaction. Bucket 0's whole range is below the floor, so
744        // its filter returns Remove; bucket 1 straddles above it.
745        schema.set_pruning_floor(transaction_bitmap::TX_BUCKET_SIZE);
746        db.compact_range_cf(transaction_bitmap::NAME, None, None)
747            .unwrap();
748
749        assert!(
750            schema
751                .get_transaction_bitmap(dim.clone(), 0)
752                .unwrap()
753                .is_none(),
754            "fully-pruned bucket 0 should be evicted by compaction",
755        );
756        assert!(
757            schema.get_transaction_bitmap(dim, 1).unwrap().is_some(),
758            "bucket 1 above the floor must remain",
759        );

892            retention_epochs: 1,
893            interval_ms: 1,

903                .unwrap_or_default()

905        };

1197            (0u64, [10u8; 32]),
1198            (500_000u64, [11u8; 32]),
1199            (999_999u64, [12u8; 32]),
1200        ];
1201        let mut batch = db.batch();
1202        for (tx_seq, digest_bytes) in entries {
1203            let digest = TransactionDigest::new(digest_bytes);
1204            batch
1205                .put(
1206                    &schema.tx_metadata_by_seq,
1207                    &U64Be(tx_seq),
1208                    &tx_metadata_by_seq::store(&tx_metadata_by_seq::Metadata {
1209                        digest,
1210                        checkpoint_seq: tx_seq,
1211                        ckpt_position: 0,
1212                        event_count: 0,
1213                        timestamp_ms: 0,
1214                    }),
1215                )
1216                .unwrap();
1217            batch
1218                .put(
1219                    &schema.tx_seq_by_digest,
1220                    &tx_seq_by_digest::Key(digest),
1221                    &U64Varint(tx_seq),
1222                )
1223                .unwrap();
1224        }
1225        batch.commit().unwrap();
1226
1227        // No prior pruning watermark (floor unknown -> 0); prune through
1228        // checkpoint 0 / tx_seq 600_000 exclusive. Only the two rows
1229        // below 600_000 are unindexed; the one at 999_999 survives.
1230        prune_history_cohort(&db, &schema, 0, 600_000).unwrap();
1231
1232        assert!(schema.get_tx_metadata_by_seq(0).unwrap().is_none());
1233        assert!(schema.get_tx_metadata_by_seq(500_000).unwrap().is_none());
1234        assert!(schema.get_tx_metadata_by_seq(999_999).unwrap().is_some());
1235        assert!(
1236            schema
1237                .get_tx_seq_by_digest(&TransactionDigest::new([10u8; 32]))
1238                .unwrap()
1239                .is_none()
1240        );
1241        assert!(
1242            schema
1243                .get_tx_seq_by_digest(&TransactionDigest::new([11u8; 32]))
1244                .unwrap()
1245                .is_none()
1246        );
1247        assert!(
1248            schema
1249                .get_tx_seq_by_digest(&TransactionDigest::new([12u8; 32]))
1250                .unwrap()
1251                .is_some()
1252        );
1253        assert_eq!(
1254            schema.get_pruning_watermarks().unwrap(),
1255            Some(Watermarks {
1256                tx_seq_lo: 600_000,
1257                checkpoint_lo: 1,
1258            }),
1259        );
1260    }
